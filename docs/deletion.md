# 削除仕様

## 基本方針

- 投稿削除とコメント削除は、退避テーブルへ保存してから通常テーブルから削除する
- チャンネル削除とフォーラム削除は、配下データも含めて物理削除する
- 個別の投稿・コメント削除で退避したデータだけを 30 日保持し、`npm run db:purge` で完全削除する

## 投稿削除

- 対象:
  - 投稿本体
  - 投稿添付
  - 配下コメント
  - 配下コメント添付
- 挙動:
  - `DeletedPost` / `DeletedComment` / `DeletedAttachment` に退避してから削除する

## コメント削除

- 対象:
  - コメント本体
  - コメント添付
- 挙動:
  - `DeletedComment` / `DeletedAttachment` に退避してから削除する

## チャンネル削除

- 対象:
  - チャンネル本体
  - 配下投稿
  - 配下コメント
  - 配下添付
- 挙動:
  - すべて物理削除する
  - 退避テーブルには保存しない

## フォーラム削除

- 対象:
  - フォーラム本体
  - 配下チャンネル
  - 配下投稿
  - 配下コメント
  - 配下添付
  - 参加者関連付け
  - 招待
- 挙動:
  - すべて物理削除する
  - 退避テーブルには保存しない

## purge バッチ

- 実行コマンドは `npm run db:purge`
- 対象は個別の投稿削除・コメント削除で退避された `DeletedPost` / `DeletedComment` / `DeletedAttachment`
- チャンネル削除とフォーラム削除で消えたデータは purge 対象にならない

## 実動作テスト

- `npm run db:test-deletion`
- 確認内容:
  - コメント削除は退避される
  - 投稿削除は退避される
  - チャンネル削除は物理削除される
  - フォーラム削除は物理削除される
  - purge は退避済みデータだけを完全削除する
