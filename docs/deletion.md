# 削除仕様

## 目的

- 削除済みデータが通常画面に表示される事故を防ぐ
- 削除後も一定期間は調査可能な状態で保持する
- 保持期限経過後は完全に削除する

## 基本方針

- 通常テーブルには削除済みデータを残さない
- 削除時は、削除対象を退避テーブルへ保存してから通常テーブルから物理削除する
- 退避データの保持期間は 30 日間とする
- 保持期限を過ぎた退避データと添付ファイルは完全削除する

## 削除対象

### コメント削除

コメント削除時は、以下を削除対象とする。

- コメント本体
- コメントに紐づく添付ファイル

### 投稿削除

投稿削除時は、以下を削除対象とする。

- 投稿本体
- 投稿に紐づく添付ファイル
- 投稿に紐づくコメント本体
- 各コメントに紐づく添付ファイル

投稿削除時に、配下コメントだけを残すことはしない。

## 削除処理の流れ

削除処理は必ず 1 トランザクションで行う。

### コメント削除

1. コメント添付を `DeletedAttachment` に退避する
2. コメント本体を `DeletedComment` に退避する
3. 通常テーブルのコメント添付を削除する
4. 通常テーブルのコメント本体を削除する

### 投稿削除

1. 投稿配下のコメント添付を `DeletedAttachment` に退避する
2. 投稿配下のコメントを `DeletedComment` に退避する
3. 投稿添付を `DeletedAttachment` に退避する
4. 投稿本体を `DeletedPost` に退避する
5. 通常テーブルのコメント添付を削除する
6. 通常テーブルのコメントを削除する
7. 通常テーブルの投稿添付を削除する
8. 通常テーブルの投稿本体を削除する

## 退避テーブルの保持方針

- `DeletedPost`
- `DeletedComment`
- `DeletedAttachment`

各退避データは以下を持つ。

- 元データの ID
- 元の親 ID
- 削除日時
- 削除実行者 ID
- 完全削除予定日時 (`purgeAfter`)

`purgeAfter` は削除日時から 30 日後とする。

## 添付ファイルの削除方針

- 添付ファイル本体はローカルストレージに保存する
- 削除時点ではファイル本体を即時削除しない
- 保持期間中は復旧調査用に保持する
- `purgeAfter` 到達後、退避レコードと対応するファイル本体を削除する

## purge バッチ

- 定期実行バッチで完全削除を行う
- 実行コマンドは `npm run db:purge`
- 実行頻度は 1 日 1 回を想定する
- `DeletedAttachment` の期限切れレコードを削除する前に、対応するローカルファイルを削除する
- その後 `DeletedComment` と `DeletedPost` の期限切れレコードを削除する

## 実動作テスト

- 削除処理と purge をまとめて検証するテストコマンドとして `npm run db:test-deletion` を用意する
- このテストではコメント削除、投稿削除、添付退避、期限切れ purge、ローカルファイル削除を確認する

## 無効参照の扱い

- 削除済み添付ファイルへの `attachment:...` 参照は解決しない
- 削除済み投稿・コメントは通常画面に表示しない

## 初期実装の範囲

初期実装では以下を対象とする。

- コメント削除
- 投稿削除
- 退避データの 30 日保持
- 保持期限経過後の完全削除

初期実装では以下は対象外とする。

- 削除データの復元
- 削除理由の入力
- 削除履歴の画面表示
