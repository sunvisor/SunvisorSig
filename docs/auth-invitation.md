# 認証・招待仕様

## 目的

- 新規ユーザーだけをメール招待経由で登録できるようにする
- 既存ユーザーの別フォーラム参加は、管理画面の参加者管理で行う
- 招待と認証の責務を分け、後からログイン機能を追加しやすくする

## 基本方針

- 招待は新規アカウント作成時だけ使う
- 既存アカウントのフォーラム追加には招待を使わない
- 招待送信時点では `User` を作成しない
- アクティベーション完了時に `User` と `ForumMember` を作成する

## 役割分担

### Invitation

- 新規ユーザーへの招待イベントを表す
- 招待メール送信、期限、利用済み状態を持つ
- どのフォーラムへ、どのロールで参加するかを持つ

### User

- アクティベーション完了後の実ユーザーを表す
- 表示名、メールアドレス、パスワードなどを持つ
- フォーラム参加そのものは持たず、`ForumMember` 経由で参加する

### ForumMember

- どのユーザーがどのフォーラムに参加しているかを表す
- 招待中の状態は持たない
- 実際にアクティベーションが完了した後に作成する

## 招待の対象

- 招待対象は未登録メールアドレスのみとする
- 同じメールアドレスの `User` が既に存在する場合、招待は作成しない
- 既存ユーザーをフォーラムへ参加させたい場合は、管理画面の参加者追加で対応する

## 招待フロー

1. 管理者がフォーラム設定画面からメールアドレスを入力して招待を作成する
2. システムが `Invitation` を作成し、招待メールを送る
3. 利用者がメール内リンクからアクティベーションページへアクセスする
4. 利用者が表示名とパスワードを設定する
5. システムが `User` を作成する
6. システムが `ForumMember` を作成する
7. システムが `Invitation` を利用済みにする
8. 利用者はログインして利用可能になる

## アクティベーションページで入力させる項目

- 表示名
- パスワード

初期段階ではこれだけに絞る。
プロフィール詳細やアイコン設定は後回しにする。

## Invitation の状態

- `PENDING`
- `ACCEPTED`
- `EXPIRED`
- `CANCELED`

### PENDING

- 招待作成直後
- まだ利用されていない

### ACCEPTED

- アクティベーションが完了した状態
- `User` と `ForumMember` が作成済み

### EXPIRED

- 有効期限切れ
- リンクは無効

### CANCELED

- 管理者が取り消した状態
- リンクは無効

## Invitation に必要な情報

- `id`
- `forumId`
- `email`
- `role`
- `token`
- `status`
- `expiresAt`
- `acceptedAt`
- `canceledAt`
- `createdByUserId`
- `createdAt`

## トークンの扱い

- 招待リンクにはランダムで十分長いトークンを含める
- トークンは推測困難である必要がある
- 1つの招待に対して1つのトークンを発行する
- 利用済み、期限切れ、取消済みのトークンは無効とする

## 既存ユーザー招待時の扱い

- `User.email` と一致するメールアドレスが既に存在する場合は招待を作成しない
- 管理画面で「既存ユーザーです。参加者管理から追加してください」と案内する

## ForumMember 作成タイミング

- 招待送信時には作成しない
- アクティベーション完了時に作成する

この方針により、招待中ユーザーと実参加者を明確に分ける。

## MVP の実装順

1. `Invitation` エンティティを Prisma schema に追加する
2. 管理画面に招待作成フォームを追加する
3. アクティベーションページを追加する
4. `User` 作成と `ForumMember` 作成を実装する
5. ログイン機能を追加する

## 初期実装でやらないこと

- 既存ユーザーへの招待
- 招待メールの再送
- 複数フォーラムへの同時招待
- パスワード再設定
- 招待履歴の詳細画面
