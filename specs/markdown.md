# Markdown 仕様

## 目的

投稿とコメントの本文は Markdown で保存する。
添付ファイルへの参照は、通常の URL ではなく独自記法で表現する。

## 本文の保存形式

- 本文は Markdown 文字列をそのまま保存する
- HTML の直接埋め込みは許可しない
- 外部 URL の画像埋め込みは許可しない

## 添付ファイル参照の独自記法

添付ファイルの参照には次の記法を使う。

```md
[alt](attachment:originalFileName)
```

例:

```md
[見積書](attachment:quote.pdf)
[画面イメージ](attachment:screen.png)
```

## 解決ルール

- `attachment:` の後ろには、添付ファイルの参照名を指定する
- 投稿本文では、その投稿に添付されたファイルだけを参照できる
- コメント本文では、そのコメントに添付されたファイルだけを参照できる
- 参照名が一致する添付ファイルが存在する場合のみ、有効なリンクとして解決する
- 一致する添付ファイルが存在しない場合は、無効なリンクとして扱う

## 添付ファイル名の重複ルール

同じ投稿または同じコメント内で、同名ファイルがアップロードされた場合は自動で連番を付ける。

例:

- 1つ目: `report.pdf`
- 2つ目: `report(2).pdf`
- 3つ目: `report(3).pdf`

ルール:

- 連番は拡張子の直前に付ける
- `(1)` は使わない
- 重複判定は投稿単位またはコメント単位で行う
- 確定したファイル名を Markdown の参照名として使う

## 運用ルール

- 添付後のファイル名変更は許可しない
- 本文中の `attachment:...` は、確定済みの添付ファイル名と一致している必要がある
- 削除された添付ファイルへの参照は解決しない
